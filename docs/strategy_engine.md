# 交易系统指标

## 线段生成

依靠charting_library图表库，其自带指标引擎，也就是说会从左往右对每个K线执行。

指标作用：当价格上穿EMA20且和前面下跌段间隔5根K线以上，构建上涨段；当价格下穿EMA20且和前面上涨段间隔5根K线以上，构建下跌段。

当K线没有EMA20时候，不执行，只有K线有EMA20时候才开始执行。起点K线收盘价若低于EMA20，则认为前面是下跌段，下跌段的终点则是该K线最低价；起点K线收盘价若高于EMA20，则认为前面是上涨段，上涨段的终点则是该K线最高价。当然实际情况，这个认为的线段是不必画出的，只是作为计算的参照。

假设前面线段是下跌段，从下跌段终点K线往右执行遍历。

- 若新K线最低价低于EMA20且价格低于下跌段终点，则下跌段终点移动到该K线最低价。
- 若新K线最低价低于EMA20但价格高于下跌段终点，不操作。
- 若新K线最高价高于EMA20但和下跌段终点间隔5根K线以内，不操作，下跌段维持。
- 若新K线最高价高于EMA20但和下跌段终点间隔5根及5根以上，下跌段结束，开始画上涨段，上涨段的起点就是前面下跌段的终点K线最低价，上涨段终点则为当前K线最高价。

在遍历过程中，会维护一个线段数组，这个是根据K线更新的。

## 动能衰竭判断

动能衰竭判断的比较对象：上涨趋势中，要比较相邻两个上涨段；下跌趋势中，比较相邻两个下跌段。

力度定义：线段价差越大说明力度越强，线段跨越时间越短说明力度越强。所以力度大概就是线段价差和跨越时间的比值。我觉得力度需要一个很好的定义，分子肯定是线段价差，但分母是时间跨度还是K线跨度，不太确定，最好能明显看出。由于有时候线段端点并非最高最低价，所以取线段内包括端点的最高最低价格计算即可。

动能衰竭标注：上涨趋势中，新的上涨段比旧上涨段力度小；下跌趋势中，新的下跌段比旧下跌段力度小。中间态，看先前趋势是啥，那么维持到这里的趋势状态就是啥。

趋势的基础定义：线段的高点抬高、低点也抬高，此为上涨趋势；或者线段的高点变低，低点也变低，此为下跌趋势。所以一般来说至少涉及4个点。而高点抬高低点变低，或者高点变低低点抬高，属于是中间态。

> 趋势是偏向于维持的。当前是上涨趋势，那么经过中间态，后续仍旧是高低点同时抬高，那么仍旧维持上涨趋势。当前是上涨趋势，经过中间态，后续变成高低点同时变低，那么则转化为了下跌趋势。

算法的一开始必须至少有3条线段，头3条线段如果符合趋势定义，那么当前就确定为上涨/下跌趋势。头3条线段如果不符合趋势定义，那么往后找，直到找到第一个符合趋势定义的。

假设当前到达第3段已经确定是下跌趋势，那么比较第1段和第3段力度，若第3段力度更弱，则标注；若更强，则不标准。

那么第4段肯定就是上涨段了，到达第4段若仍符合趋势定义，是不比较的，因为在下跌趋势中第2和第4都是上涨段。若不符合下跌趋势定义，那么说明是中间态了，也不比较。

到了第5段，就是下跌段了，假设前面是下跌趋势经过了中间态，此时第5段就很关键了。哪怕是经过中间态，只要没有符合上涨趋势的4个点，就仍旧是下跌趋势。所以此时仍旧是下跌趋势，比较第3段和第5段，此时有两种可能，构成高低点变低的下跌趋势，那么就按照动能衰竭判断即可；构成中间态，那么也是按照动能衰竭判断。
