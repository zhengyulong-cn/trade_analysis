# 交易系统指标

## 线段生成

依靠charting_library图表库，其自带指标引擎，也就是说会从左往右对每个K线执行。

指标作用：当价格上穿EMA20且和前面下跌段间隔5根K线以上，构建上涨段；当价格下穿EMA20且和前面上涨段间隔5根K线以上，构建下跌段。

当K线没有EMA20时候，不执行，只有K线有EMA20时候才开始执行。起点K线收盘价若低于EMA20，则认为前面是下跌段，下跌段的终点则是该K线最低价；起点K线收盘价若高于EMA20，则认为前面是上涨段，上涨段的终点则是该K线最高价。当然实际情况，这个认为的线段是不必画出的，只是作为计算的参照。

假设前面线段是下跌段，从下跌段终点K线往右执行遍历。

- 若新K线最低价低于EMA20且价格低于下跌段终点，则下跌段终点移动到该K线最低价。
- 若新K线最低价低于EMA20但价格高于下跌段终点，不操作。
- 若新K线最高价高于EMA20但和下跌段终点间隔5根K线以内，不操作，下跌段维持。
- 若新K线最高价高于EMA20但和下跌段终点间隔5根及5根以上，下跌段结束，开始画上涨段，上涨段的起点就是前面下跌段的终点K线最低价，上涨段终点则为当前K线最高价。

在遍历过程中，会维护一个线段数组，这个是根据K线更新的。

## 动能衰竭判断

判断动能衰竭的时机：MACD指标的DIFF和DEA出现金叉或死叉。以金叉或死叉位置往前找最近的3条本级别线段，然后识别这3条线段a-b-c的结构。

对于金叉时机，说明是要判断下跌衰竭的，所以a、c必须是下跌段，b是上涨段，以此来判断c相对于a是否下跌动能衰竭。

对于死叉时机，说明是要判断上涨衰竭的，所以a、c必须是上涨段，b是下跌段，以此来判断c相对于a是否上涨动能衰竭。

判断上涨衰竭，在a、c都是上涨段，b是下跌段的基础上，进行如下判定：

- `a.low<=c.low && a.high<c.high`：说明上涨段c突破上涨段a的高点，此时需要进行MACD面积比较。当c段面积比a段小，那么久构成上涨动能衰竭。
- `a.low<=c.low && a.high>=c.high`：说明上涨段c没有突破上涨段a的高点，此时肯定构成上涨动能衰竭。
- `a.low>c.low && a.high<c.high`：说明上涨段c包裹整个上涨段a，此时肯定不构成衰竭。
- `a.low>c.low && a.high>=c.high`：说明判断结构错误。

对于`a.low<=c.low && a.high<c.high`的情况，要结合MACD进行判断，MACD的SHORT默认为4，LONG默认为20，MID默认为20。

- 上涨段：以`[start.index, end.index]`区间，统计内部正柱面积总和。上涨段是不计入负柱的。

- 下跌段：以`[start.index, end.index]`区间，统计内部负柱的面积总和，取绝对值。下跌段是不计入正柱的。

> 注意区间统计左右都是闭区间，两侧对应的MACD面积都是算的。
